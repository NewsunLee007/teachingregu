# 界面美化

## 目标

在不破坏现有功能的前提下，对页面进行可访问性优先的视觉与信息架构重构，重点包括：

- 标题体系（H1-H6）分级清晰：字号、字重、颜色与字体族有显著差异
- 日间/夜间模式对比度达标（WCAG 2.1 AA）：日间 ≥ 4.5:1，夜间 ≥ 7:1
- 目录结构精修：一级目录聚合为固定集合，移除冗余装饰符
- 交互状态机梳理：一级目录锁定高亮，二级展开；再次点击一级可折叠
- 无障碍完善：aria-level、aria-expanded、键盘导航顺序与视觉顺序一致

## 工作方式

1. 先识别现有 DOM 结构中“一级目录/二级目录”的实际来源与生成逻辑（通常来自渲染函数与 heading 解析器）。
2. 将目录组件抽象为统一的可复用 DOM 结构：
   - 主目录（一级）：纵向列表
   - 子目录（二级及以下）：横向标签栏或折叠面板
   - 目录容器与样式类名统一，且在 DOM 上保持相邻兄弟节点，便于维护
3. 在 CSS 中建立 token 化的主题变量与 typography scale：
   - H1-H6 各自独立 font-family、font-size、font-weight、color
   - 通过 CSS 变量统一管理，并为 dark.css 提供覆盖变量
4. 夜间模式实现遵循优先级：
   - prefers-color-scheme: dark 默认启用夜间
   - 用户手动切换写入 localStorage，并覆盖系统偏好
5. 为交互与无障碍编写可测试的状态与属性：
   - 所有可展开节点必须同步 aria-expanded
   - 目录节点必须写入 aria-level
   - Tab 顺序遵循视觉：上→下、左→右

## 验收要点清单

- 一级目录文本不含“——”
- 夜间模式主目录文字为 #FFFFFF 或 #E0E0E0 且 opacity ≥ 0.9
- 锚点滚动误差 < ±2px（使用 scroll-margin-top + 精确定位）
- 键盘可达性：Tab/Enter/Space 可操作目录与按钮
